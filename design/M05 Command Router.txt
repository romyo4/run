# M05 Command Router

> **Design Freeze 注記**: Design Freeze監査により、Scheduler(M14)との相互依存(STATUSコマンドのCommand Router→Scheduler転送)を解消した。依存方向はScheduler→Command Routerの一方向のみとし、ステータス取得はState Manager経由に変更した。経緯は`CHANGELOG.md`参照。

## 1. Overview

Command Router は、AI Development Pipeline において、ユーザーから受信した自然言語コマンドを適切なモジュールへ振り分けるモジュールである。

本モジュールの目的は、入力されたコマンドを解析し、実行対象となるモジュールを決定し、処理要求を転送することである。

Command Router は**コマンドの振り分けのみ**を担当し、要件分析・設計・実装・レビューなどの業務処理は行わない。

### 適用対象

- Slack
- Discord
- CLI
- Web UI
- API
- Scheduler（内部コマンド）

### 対象外

- 要件分析
- Task生成
- Workflow生成
- コード生成
- GitHub操作
- 権限判定
- 実処理

---

# 2. Responsibilities

## 2.1 担当

Command Router は以下を担当する。

- コマンド受信
- 入力形式の正規化
- コマンド種別判定
- 実行対象モジュール決定
- 実行要求の転送
- 不正コマンドの検出

---

## 2.2 担当しない

以下は本モジュールの責務外とする。

- Plannerによる要件分析
- Designerによる設計
- Executorによる実装
- Reviewerによるレビュー
- Scheduler管理
- Permission判定
- Configuration取得

---

# 3. Design

## 3.1 入力

Command Router が受け付ける入力は以下とする。

```text
command_id
source
user_id
timestamp
command
attachments
metadata
```

---

## 3.2 正規化

入力元ごとの差異を吸収し、共通フォーマットへ変換する。

例

Slack

```
/plan LP改善
```

Discord

```
@bot plan LP改善
```

↓

共通形式

```text
command_type = PLAN

payload = "LP改善"
```

---

## 3.3 Command Type

MVPでは以下をサポートする。

|Command|転送先|
|--------|------|
|PLAN|Planner|
|DESIGN|Designer|
|IMPLEMENT|Executor|
|REVIEW|Reviewer|
|STATUS|State Manager|
|HELP|System|

STATUSはSchedulerへは転送しない。State Managerから直接状態を取得することで、Command Router→Scheduler方向の依存を発生させない(Scheduler→Command Routerの一方向依存のみを許可する)。

Command追加時はRouterのみ更新する。

---

## 3.4 Routing

Routingテーブル

|Command|Destination|
|--------|-----------|
|PLAN|Planner|
|DESIGN|Designer|
|IMPLEMENT|Executor|
|REVIEW|Reviewer|
|STATUS|State Manager|

Router自身は処理内容を理解しない。Scheduler宛の転送は行わない(依存関係は Scheduler→Command Router の一方向のみとする)。

---

## 3.6 エラー処理

認識できないCommandは

```text
UNKNOWN
```

として扱う。

UNKNOWNは

- 実行しない
- ログ出力
- ユーザーへ通知

を行う。

# 4. Constraints

## 4.1 Routerは判断しない

Command Router は

- 要件分析
- 優先順位判断
- 設計判断
- コード生成

を行ってはならない。

---

## 4.2 Routerは状態を持たない

Workflow

Task

Knowledge

などの状態を保持してはならない。

---

## 4.3 Routerは同期処理

MVPでは

同期処理のみとする。

Queueによる非同期処理は採用しない。

---

## 4.4 Routerは転送専用

Router自身がGitHub APIを呼び出してはならない。

Slack返信を生成してはならない。

PRを作成してはならない。

---

## 4.5 Logging

以下を記録する。

```text
timestamp

source

command

command_type

destination

result
```

Secretは出力しない。

---

**判定：実装可能**

---

# 6. Summary

Command Router は、AI Development Pipeline の入口となるモジュールである。

責務は**「受信・正規化・振り分け・転送」**に限定し、要件分析や実装処理には関与しない。

MVPでは **Command Type → Destination Module** のシンプルなルーティング方式を採用し、システム全体の疎結合と拡張性を確保する。
