# M08 Design Auditor

## 1. Overview

Design Auditor は、AI Development Pipeline において、Architect が作成した Design Document を監査し、設計品質を保証するモジュールである。

本モジュールの目的は、設計書を実装へ渡す前に、要求との整合性・設計品質・MVP適合性・アーキテクチャ整合性を確認することである。

Design Auditor は**設計監査のみ**を担当し、設計の修正・コード生成・Pull Request 作成は行わない。

### 適用対象

- Architecture Design
- Module Design
- API Design
- Data Structure Design
- Refactoring Design

### 対象外

- 要件分析
- システム設計
- コード生成
- テスト実装
- Pull Request作成
- GitHub操作

---

# 2. Responsibilities

## 2.1 担当

Design Auditor は以下を担当する。

- Design Document監査
- 要件との整合性確認
- Architecture整合性確認
- MVP適合性確認
- 責務分離確認
- 重複設計検出
- 設計品質レポート作成
- Architectへの差し戻し判定

---

## 2.2 担当しない

以下は本モジュールの責務外とする。

- 要件分析
- 設計書修正
- コード生成
- Pull Request作成
- コードレビュー
- Workflow実行

---

# 3. Design

## 3.1 入力

Design Auditor が受け付ける入力。

```text
workflow_id
execution_plan
design_document
knowledge
architecture_guidelines
```

---

## 3.2 監査項目

Design Auditor は以下を監査する。

- 要件充足
- 責務分離
- モジュール境界
- Interface整合性
- Domain整合性
- Configuration整合性
- MVP適合性
- 再利用可能性
- 過剰設計の有無

---

## 3.3 監査結果

監査結果は以下のいずれかとする。

```text
PASS

PASS_WITH_COMMENT

REWORK_REQUIRED

REJECT
```

---

## 3.4 Audit Report

生成する成果物。

```text
Audit ID

Result

Findings

Warnings

Violations

Recommendations
```

---

## 3.5 公開インターフェース

入力

```text
Design Document
```

出力

```text
Audit Report
```

---

入力

```text
Design Document
```

出力

```text

```

---

入力

```text

```

# 4. Constraints

## 4.1 Auditorは設計しない

Design Auditor は

- Module設計
- API設計
- Data設計

を行ってはならない。

---

## 4.2 Auditorは修正しない

設計書を直接修正してはならない。

修正が必要な場合は Architect へ差し戻す。

---

## 4.3 Auditorは実装しない

- コード生成
- GitHub操作
- Pull Request作成

を行ってはならない。

---

## 4.4 監査基準固定

監査基準は Foundation に準拠する。

監査中に独自ルールを追加してはならない。

---

## 4.5 Logging

以下を記録する。

```text
timestamp

workflow_id

design_id

audit_result

finding_count

warning_count

result
```

Secretは出力しない。

---

# 5. Design Audit

## 5.1 責務確認

- 設計監査のみ
- Audit Report作成のみ

設計・実装を含まない。

**判定：OK**

---

## 5.2 Foundation整合性

**F00**

- Design Before Code
- Quality
- Simplicity
- Traceability

に適合。

**判定：OK**

---

**F01**

Design Domainを変更しない。

**判定：OK**

---

**F02**

Common Interface経由で利用可能。

**判定：OK**

---

**F03**

Configurationを参照可能。

**判定：OK**

---

## 5.3 重厚壮大化監査

以下はMVP対象外。

- AI設計生成
- 自動修正
- UML生成
- コスト最適化
- パフォーマンス解析
- セキュリティ自動修正
- Enterprise Design Governance

**判定：削除済み**

---

## 5.4 実装可能性

Design Auditor は Design Document を監査し、監査結果を返す単一責務のモジュールとして実装可能である。

設計品質を保証した後にのみ Executor へ引き渡す。

**判定：実装可能**

---

# 6. Summary

Design Auditor は AI Development Pipeline の設計品質保証モジュールである。

責務は**「Design Document を監査し、実装へ進めてよいかを判定すること」**に限定する。

設計品質・MVP適合性・アーキテクチャ整合性を確認し、問題がある場合は Architect へ差し戻すことで、**「Design Before Code」** の原則を実現する。
