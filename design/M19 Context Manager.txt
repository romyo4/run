# M19 Context Manager

## 1. Overview

Context Manager は、AI Development Pipeline において、各AI（Codex・Fable等）へ渡す Context を生成するモジュールである。

本モジュールの目的は、Workflow の種類に応じて必要な情報だけを収集・選択・組み立て、AIへ最適な Context を提供することである。

Context Manager は**Context の生成および提供のみ**を担当し、Knowledge の管理・Repository の解析・AI の実行は行わない。

---

## 適用対象

- Planner Context
- Architect Context
- Executor (Codex) Context
- Reviewer Context
- Weekly Reviewer (Fable) Context

---

## 対象外

- Knowledge管理
- Configuration管理
- Repository解析
- GitHub操作
- コード生成
- AI実行
- Workflow管理

---

# 2. Responsibilities

## 2.1 担当

Context Manager は以下を担当する。

- Context生成
- Context選択
- Context組み立て
- Context検証
- Context提供

---

## 2.2 担当しない

以下は本モジュールの責務外とする。

- Knowledge管理
- Configuration管理
- Repository解析
- コード生成
- AI実行
- Workflow制御

---

# 4. Constraints

## 4.1 Contextのみ生成する

Context Manager は

- コード生成
- AI実行
- Repository解析
- Workflow実行

を行ってはならない。

---

## 4.2 必要最小限のContext

Workflow に必要な情報のみを含める。

不要な情報を追加してはならない。

---

## 4.3 Repositoryを解析しない

Repository の解析・検索・要約は GitHub Manager が担当する。

Context Manager は提供された Repository Context のみを利用する。

---

## 4.4 Knowledgeを変更しない

Knowledge は参照のみとする。

内容を書き換えてはならない。

---

## 4.5 Configurationを変更しない

Configuration は参照のみとする。

内容を書き換えてはならない。

---

## 4.6 Logging

以下を記録する。

```text
timestamp

workflow_id

workflow_type

context_version

context_size

validation_result
```

Context本文・Knowledge本文・Repository情報・Secret はログへ出力してはならない。

---

# 5. Design Audit

## 5.1 責務確認

- Context生成のみ
- Context選択のみ
- Context提供のみ

Knowledge管理・Configuration管理・Repository解析・AI実行を含まない。

**判定：OK**

---

## 5.2 Foundation整合性

**F00**

- Single Responsibility
- Business First
- Context Optimization

に適合。

**判定：OK**

---

**F01**

Context Domain を利用する。

**判定：OK**

---

**F02**

Common Interface を利用する。

**判定：OK**

---

**F03**

Knowledge Manager・Configuration Manager・GitHub Manager を唯一の参照元として利用する。

**判定：OK**

---

## 5.3 重厚壮大化監査

以下はMVP対象外。

- RAG
- Embedding Search
- Vector Database
- Semantic Search
- AI Context Compression
- Token Optimization
- Long-term Memory
- Multi-Agent Context Sharing

**判定：削除済み**

---

## 5.4 実装可能性

Context Manager は Knowledge Manager・Configuration Manager・GitHub Manager が提供する情報を組み合わせ、Workflow ごとに必要最小限の Context を生成する単一責務のモジュールとして実装可能である。

Repository解析や Knowledge管理を含まないため、責務が明確でありMVPとして実装しやすい。

**判定：実装可能**

---

# 6. Summary

Context Manager は AI Development Pipeline の Context 組み立てモジュールである。

責務は**「各AIに必要な情報だけを選択・組み立て、一貫した Context を提供すること」**に限定する。

各AIは同じ Knowledge・Configuration・Repository情報を共有しながらも、Workflowごとに最適化された Context を受け取ることで、不要な情報を排除し、一貫性のある判断と効率的な処理を実現する。
