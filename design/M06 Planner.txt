# M06 Planner

## 1. Overview

Planner は、AI Development Pipeline において、ユーザーから受け取った自然言語の要求を分析し、実行可能な開発計画（Execution Plan）へ変換するモジュールである。

本モジュールの目的は、曖昧な要求を明確化し、実装可能なタスクへ分解して後続モジュールへ引き渡すことである。

Planner は**計画の作成のみ**を担当し、設計・実装・レビューは行わない。

### 適用対象

- 新規開発要求
- 機能追加要求
- 改善要求
- バグ修正要求
- リファクタリング要求

### 対象外

- システム設計
- コード生成
- テスト実装
- Pull Request作成
- レビュー
- GitHub操作

---

# 2. Responsibilities

## 2.1 担当

Planner は以下を担当する。

- 要求分析
- 要件抽出
- 曖昧な要求の補完
- 実行単位への分解
- 優先順位付け
- Execution Plan作成

---

## 2.2 担当しない

以下は本モジュールの責務外とする。

- システム設計
- モジュール設計
- コード生成
- Pull Request作成
- コードレビュー
- Workflow実行

---

# 3. Design

## 3.1 入力

Planner が受け付ける入力。

```text
workflow_id
command
normalized_request
knowledge
project_context
```

---

## 3.2 要求分析

Planner は以下を抽出する。

- 目的
- 背景
- 制約
- 成果物
- 優先度

例

入力

```text
昨日のLPのファーストビューを改善してPRを作って
```

抽出結果

```text
目的
LP改善

成果物
Pull Request

対象
ファーストビュー

制約
既存デザインを維持
```

---

## 3.3 Task分解

Planner は要求を実行単位へ分解する。

例

```text
Task1
現状分析

Task2
改善案作成

Task3
改善案評価

Task4
実装

Task5
PR作成
```

Taskは後続モジュールが処理可能な粒度とする。

---

## 3.4 優先順位

TaskごとにPriorityを付与する。

```text
HIGH

MEDIUM

LOW
```

依存関係を考慮してExecution順序を決定する。

---

## 3.5 Execution Plan

Planner が生成する成果物。

```text
Plan ID

Objective

Task List

Priority

Dependencies

Expected Output
```

---

```text
Normalized Request
```


---

## 3.7 処理フロー

```text
Command Router
        │
        ▼
Requirement Analysis
        │
        ▼
Task Decomposition
        │
        ▼
```

# 4. Constraints

## 4.1 Plannerは設計しない

Planner は

- クラス設計
- API設計
- DB設計

を行ってはならない。

---

## 4.2 Plannerは実装しない

Planner は

- コード生成
- GitHub操作
- Pull Request作成

を行ってはならない。

---

## 4.3 Plannerはレビューしない

Planner は

改善案の採否を決定しない。

評価はReviewerの責務である。

---

## 4.4 Knowledge利用

Planner は Knowledge を参照してよい。

Knowledgeを書き換えてはならない。

---

## 4.5 Logging

以下を記録する。

```text
timestamp

workflow_id

objective

task_count

execution_plan_id

result
```

Secretは出力しない。

---

- Execution Plan作成のみ

設計・実装・レビューを含まない。

## 5.3 重厚壮大化監査

- リソース配分AI
- ガントチャート生成
- マルチプロジェクト最適化

**判定：削除済み**

---

## 5.4 実装可能性

Planner は自然言語要求を Execution Plan に変換する単一責務のモジュールとして実装可能である。

Task分解・優先順位付け・Execution Plan生成を担当し、後続の Designer がその成果物を入力として利用する。

**判定：実装可能**

---

# 6. Summary

Planner は AI Development Pipeline の計画策定モジュールである。

責務は**「要求分析・Task分解・優先順位付け・Execution Plan生成」**に限定し、設計・実装・レビューには関与しない。

MVPではシンプルな Execution Plan を生成することに集中し、後続モジュールとの責務分離を維持する。
