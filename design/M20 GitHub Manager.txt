# M20 GitHub Manager

## 1. Overview

GitHub Manager は、AI Development Pipeline において、GitHub Repository に対する共通操作を提供するモジュールである。

本モジュールの目的は、GitHub API や Git 操作を一元化し、各モジュールが GitHub の実装詳細を意識することなく利用できるようにすることである。

GitHub Manager は**Repository 情報の取得および GitHub 操作のみ**を担当し、要件分析・設計・実装・レビュー・Workflow 制御は行わない。

---

## 適用対象

- Repository情報取得
- Branch情報取得
- Commit情報取得
- Pull Request情報取得
- File情報取得
- GitHub API操作

---

## 対象外

- コード生成
- Pull Requestレビュー
- Pull Request作成
- GitHubマージ
- Repository解析
- Workflow管理

---

# 2. Responsibilities

## 2.1 担当

GitHub Manager は以下を担当する。

- Repository情報取得
- Branch情報取得
- Commit情報取得
- Pull Request情報取得
- File情報取得
- Diff取得
- GitHub API呼び出し

---

## 2.2 担当しない

以下は本モジュールの責務外とする。

- コード編集
- Pull Request作成
- Pull Requestレビュー
- Repository解析
- Business判断
- Workflow制御

---

# 4. Constraints

## 4.1 GitHub操作のみ

GitHub Manager は

- コード生成
- コード修正
- Pull Requestレビュー
- Workflow制御

を行ってはならない。

---

## 4.2 Repositoryを解析しない

Repository の設計分析

依存関係分析

Business分析

は行わない。

取得のみ行う。

---

## 4.3 Read Only

MVPでは

Repository 情報取得を基本とする。

Repository 更新は Executor または PR Creator が担当する。

---

## 4.4 最小取得

Workflow に必要な Repository 情報のみ取得する。

Repository 全体を取得してはならない。

---

## 4.5 Logging

以下を記録する。

```text
timestamp

repository

operation

branch

pull_request

result

duration
```

Access Token・Secret・Repository 内容はログへ出力してはならない。

## 5.4 実装可能性

GitHub Manager は GitHub API を利用して Repository・Branch・Commit・Pull Request・File 情報を取得する単一責務のモジュールとして実装可能である。

Context Manager・PR Creator・Weekly Reviewer は GitHub Manager を介して Repository 情報を取得するため、GitHub API の実装が一元化される。

**判定：実装可能**

---

# 6. Summary

GitHub Manager は AI Development Pipeline の GitHub アクセスモジュールである。

責務は**「GitHub Repository に対する共通アクセスを提供し、必要な情報を各モジュールへ供給すること」**に限定する。

これにより、

- Context Manager は対象ファイル・差分情報を取得できる
- PR Creator は Pull Request 情報を参照できる
- Weekly Reviewer は今週マージされた PR を収集できる

各モジュールが GitHub API の詳細を実装する必要がなくなり、保守性・再利用性・責務分離を実現できる。
