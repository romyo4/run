# M21 Slack / Discord Connector

## 1. Overview

Slack / Discord Connector は、AI Development Pipeline において、Slack および Discord とシステムを接続するモジュールである。

本モジュールの目的は、ユーザーからの指示を受信し、システムからの通知を送信するための通信インターフェースを提供することである。

Slack / Discord Connector は**メッセージの送受信のみ**を担当し、コマンド解析・Workflow 制御・AI 実行は行わない。

---

## 適用対象

- Slack
- Discord

---

## 対象外

- Command解析
- Workflow起動
- Notification生成
- AI実行
- Business判断

---

# 2. Responsibilities

## 2.1 担当

Slack / Discord Connector は以下を担当する。

- メッセージ受信
- メッセージ送信
- 添付ファイル受信
- ユーザー識別
- チャネル識別
- プラットフォーム差異の吸収

---

## 2.2 担当しない

以下は本モジュールの責務外とする。

- コマンド解析
- 権限判定
- Workflow起動
- 通知内容生成
- AI実行
- Repository操作

---

# 4. Constraints

## 4.1 通信のみ担当する

Slack / Discord Connector は

- Command解析
- Workflow起動
- 権限判定
- AI実行

を行ってはならない。

---

## 4.2 共通フォーマットへ変換する

Slack と Discord の違いは Connector 内で吸収する。

Command Router はプラットフォームを意識してはならない。

---

## 4.3 状態を保持しない

MVPではセッション情報や会話履歴を保持しない。

受信したメッセージを共通フォーマットへ変換するのみとする。

---

## 4.4 Notificationを生成しない

通知本文は Notification モジュールが生成する。

Connector は送信のみ担当する。

---

## 4.5 Logging

以下を記録する。

```text
timestamp

platform

user_id

channel_id

event_type

result
```

メッセージ本文・添付ファイル・Access Token はログへ出力してはならない。

---

## 5.4 実装可能性

Slack / Discord Connector は Slack API および Discord API を利用してメッセージの送受信を行い、共通フォーマットへ変換する単一責務のモジュールとして実装可能である。

Command Router や Notification は Connector を経由して通信を行うため、プラットフォーム依存を排除できる。

**判定：実装可能**

# 6. Summary

Slack / Discord Connector は AI Development Pipeline の通信アダプタモジュールである。

責務は**「Slack・Discordとの通信を担当し、共通フォーマットへ変換すること」**に限定する。

これにより、

- ユーザーはスマートフォンから Slack や Discord で自然文の指示を送信できる
- Command Router はプラットフォームに依存せずコマンドを処理できる
- Notification は送信先を意識せず通知を配信できる

例えば、スマートフォンから

> 「昨日のLPのファーストビューを改善して、一番良い案を実装してPRを作って」

と Slack や Discord に送信すると、Connector が共通メッセージへ変換し、Command Router へ渡す。

また、PR 作成後は Notification が生成した

> 「PR #152 を作成しました」

というメッセージを Connector が Slack または Discord へ送信する。

これにより、チャットプラットフォームの違いを意識することなく、AI Development Pipeline 全体をスマートフォンから操作できる。
