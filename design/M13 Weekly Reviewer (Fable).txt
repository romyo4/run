# M13 Weekly Reviewer (Fable)

## 1. Overview

Weekly Reviewer (Fable) は、AI Development Pipeline において、一定期間（通常は1週間）に実施された開発成果を俯瞰的にレビューし、プロジェクト全体の方向性を評価するモジュールである。

本モジュールの目的は、個々の Pull Request の品質ではなく、**プロジェクト全体が事業目的へ向かって進んでいるか**を確認することである。

Weekly Reviewer は**週次レビューのみ**を担当し、要件分析・設計・実装・Pull Request レビュー・マージは行わない。

Fable は Weekly Reviewer のレビューエンジンとして利用する。

### 適用対象

- Weekly Review
- Pull Request Summary
- Business Goal Review
- MVP Review
- Technical Debt Review
- Priority Recommendation

### 対象外

- 要件分析
- システム設計
- コード生成
- Pull Request作成
- Pull Requestレビュー
- GitHubマージ
- リリース

---

# 2. Responsibilities

## 2.1 担当

Weekly Reviewer は以下を担当する。

- 今週マージされた Pull Request の収集
- 実装内容の要約
- Business Goalとの整合性評価
- MVP適合性評価
- 技術的負債評価
- 優先順位提案
- 週次レビュー報告書作成

---

## 2.2 担当しない

以下は本モジュールの責務外とする。

- Pull Requestレビュー
- コード修正
- コード生成
- 設計変更
- マージ
- リリース

# 4. Constraints

## 4.1 Weekly Reviewerは修正しない

Weekly Reviewer は

- コード修正
- Design修正
- Pull Request修正

を行ってはならない。

---

## 4.2 長期視点で評価する

個々のPRではなく、

プロジェクト全体

を評価対象とする。

---

## 4.3 Business Goal優先

レビューでは

```text
Business Goal

＞

MVP

＞

Architecture

＞

Coding Style
```

の順で評価する。

---

## 4.4 提案のみ行う

Weekly Reviewer は

優先順位

改善案

技術的負債

を提案する。

実施判断は Project Owner が行う。

---

## 4.5 Logging

以下を記録する。

```text
timestamp

review_period

merged_pr_count

technical_debt_count

recommendation_count

result
```

Secret・Access Token・Credential はログへ出力してはならない。

# 6. Summary

Weekly Reviewer (Fable) は AI Development Pipeline の週次プロジェクトレビュー モジュールである。

責務は**「1週間の開発成果を俯瞰し、Business Goal・MVP・技術的負債・次の優先順位を評価すること」**に限定する。

例えば、毎週以下のようなレビューを自動生成する。

- 今週マージされたPRを一覧化する
- 事業目的（例：LINE登録最大化）とズレていないか評価する
- MVPに不要な機能が紛れ込んでいないか指摘する
- 新たに生まれた技術的負債を整理する
- 来週最優先で取り組むべき改善を1～3件提案する

これにより、個々の実装品質だけでなく、**プロジェクト全体が正しい方向へ進んでいるか**を継続的に確認できる。
