# M11 PR Creator

## 1. Overview

PR Creator は、AI Development Pipeline において、Tester が品質ゲートを通過した実装成果物を GitHub Pull Request として登録するモジュールである。

本モジュールの目的は、Pull Request の作成を標準化し、タイトル・説明・変更内容・関連情報を一貫した形式で GitHub に登録することである。

PR Creator は**Pull Request の作成のみ**を担当し、実装・レビュー・マージは行わない。

### 適用対象

- GitHub Pull Request作成
- Pull Request更新
- Pull Request説明生成
- ラベル付与
- Reviewer設定

### 対象外

- 要件分析
- システム設計
- コード生成
- テスト実行
- コードレビュー
- Pull Requestマージ
- Release作成

---

# 2. Responsibilities

## 2.1 担当

PR Creator は以下を担当する。

- Pull Request作成
- Pull Requestタイトル生成
- Pull Request本文生成
- 変更ファイル一覧取得
- Reviewer設定
- Label設定
- Pull Request URL取得
- PR作成結果レポート生成

---

## 2.2 担当しない

以下は本モジュールの責務外とする。

- コード生成
- コード修正
- テスト実行
- レビュー
- マージ
- GitHub Workflow実行

---

# 3. Design

## 3.1 入力

PR Creator が受け付ける入力。

```text
workflow_id

implementation_result

test_report

repository_information

branch_information

project_context
```

---

## 3.2 Pull Request生成

PR Creator は以下を生成する。

- Title
- Description
- Summary
- Changed Files
- Related Issue
- Review Checklist

---

## 3.3 Pull Requestテンプレート

生成する本文例

```text
# Summary

## Purpose

## Changes

## Test Result

## Related Issue

## Notes
```

本文フォーマットは全PRで統一する。

---

## 3.4 成果物

PR Creator が生成する成果物。

```text
Pull Request

Pull Request URL

Pull Request Number

Creation Report
```

---

## 3.5 公開インターフェース

### create_pr()

入力

```text
Implementation Result
```

出力

```text
Pull Request
```

---

### update_pr()

入力

```text
Pull Request
```

出力

```text
Updated Pull Request
```

---

### assign_reviewers()

入力

```text
Pull Request
```

出力

```text
Assignment Result
```

---

### publish()

入力

```text
Completed Pull Request
```

出力

```text
Pull Request URL
```

---

## 3.6 処理フロー

```text
Tester
      │
      ▼
Quality Gate PASS
      │
      ▼
PR Creator
      │
      ▼
Title生成
      │
      ▼
Description生成
      │
      ▼
GitHub Pull Request
      │
      ▼
Reviewer
```

---

# 4. Constraints

## 4.1 PR Creatorは実装しない

PR Creator は

- コード生成
- コード修正
- テスト実行

を行ってはならない。

---

## 4.2 Quality Gate必須

Quality Gate が PASS していない場合、

Pull Request を作成してはならない。

---

## 4.3 Branch制約

指定された Branch のみを対象とする。

Branch の切り替えや統合は行わない。

---

## 4.4 GitHub制約

MVPでは GitHub の Pull Request のみ対応する。

GitLab・Bitbucket は対象外とする。

---

## 4.5 Logging

以下を記録する。

```text
timestamp

workflow_id

repository

branch

pull_request_number

pull_request_url

result
```

Secret・Access Token・Credential はログへ出力してはならない。

---

# 5. Design Audit

## 5.1 責務確認

- Pull Request作成のみ
- GitHub登録のみ

実装・テスト・レビューを含まない。

**判定：OK**

---

**F00**

- Automation First
- Documentation First
- Single Responsibility

に適合。

**判定：OK**

**判定：OK**

---

**F02**

Common Interface を利用する。

**判定：OK**

---

**F03**

Configuration から GitHub 接続情報を取得する。

**判定：OK**

---

## 5.3 重厚壮大化監査

以下はMVP対象外。

- 自動マージ
- Draft PR管理
- Release Note生成
- GitHub Project更新
- GitHub Issue自動生成
- Multi Repository PR
- GitHub App管理

**判定：削除済み**

---

## 5.4 実装可能性

PR Creator は Tester が品質ゲートを通過した成果物を GitHub Pull Request として登録する単一責務のモジュールとして実装可能である。

Pull Request の生成・登録・URL取得のみを担当する。

**判定：実装可能**

---

# 6. Summary

PR Creator は AI Development Pipeline の Pull Request 作成モジュールである。

責務は**「品質確認済みの成果物を GitHub Pull Request として登録すること」**に限定する。

実装・テスト・レビューとは責務を分離し、Pull Request 作成を標準化することで、一貫した開発フローを実現する。
